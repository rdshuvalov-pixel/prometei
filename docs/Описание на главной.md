# Prometei — живой сервис для поиска и “упаковки” вакансий

**Prometei** — это мой петпроект, который автоматизирует самый скучный и дорогой кусок job search: сбор вакансий из десятков источников → дедупликацию → скоринг по правилам → подготовку материалов для отклика. Цель простая: *меньше мусора, больше точных попаданий и готовых текстов, чтобы отклик занимал минуты, а не часы.*

## Что он решает (постановка задачи)

Когда вы ищете Senior-level роль, ручной мониторинг карьерных страниц и джоббордов превращается в бесконечную ленту повторов, “устарело 2 недели назад” и вакансий не по профилю. Prometei делает это как продуктовый конвейер:

- собирает вакансии из **карьерных страниц компаний** и **job boards**,
- убирает дубли по связке **company + role_title** (чтобы не жить в дежавю),
- применяет фильтры и **скоринг (0–100)** по понятным правилам,
- и для всего, что проходит порог, готовит материалы для отклика (включая 2 варианта cover letter).

## Как это работает внутри (бизнес-процесс, по шагам)

1. **Сбор** вакансий по ролям из Tier-источников (career pages + boards).
2. **Дедупликация** и сохранение в базе: “та же компания + та же роль” не размножается.
3. **Скоринг** и отсев по правилам качества (формат/локация/свежесть/релевантность).
4. **Генерация материалов**: для каждой подходящей вакансии — письма и заметки.
5. **Хранение результата**: всё живёт в базе, с понятными статусами и историей запусков.

## Что этот проект показывает про мои навыки

Если коротко - **я собрал работающий продуктовый сервис end-to-end**:

- **Product thinking:** спроектировал процесс как воронку с правилами качества, статусами и предсказуемым результатом.
- **Engineering:** web-UI + API + очередь задач + воркер + база данных (то есть не скрипт “у меня на ноуте”).
- **Data/Automation:** нормальная модель данных, дедуп, повторяемые прогоны, расписание, контроль запусков.
- **Коммуникация результата:** на выходе не “сырые ссылки”, а структурированный набор, по которому реально можно действовать.

И да — я **специально не автоматизировал отправку откликов**. Потому что если бы ещё и это прикрутил, я бы внёс вклад в ускоренное разрушение рынка найма (а я, как ни странно, в него всё ещё верю) 🙂

# Prometei — a real job search pipeline service

**Prometei** is my pet-project that automates the most boring and expensive part of job search: collecting vacancies from dozens of sources → deduplication → rule-based scoring → preparing outreach materials. The goal is simple: *less noise, more precise matches, and ready-to-send texts — so applying takes minutes, not hours.*

## The problem it solves

When you’re hunting for a senior role, manually checking company career pages and job boards turns into an endless feed of duplicates, “posted two weeks ago” listings, and roles that aren’t really relevant. Prometei turns this into a product-grade pipeline:

- pulls roles from **company career pages** and **job boards**,
- removes duplicates by **company + role title** (so you don’t live in déjà vu),
- applies clear filters and a **0–100 scoring model**,
- and for everything above the threshold, prepares outreach materials (including two cover letter variants).

## How it works (business process, step by step)

1. **Collect** roles by target titles from tiered sources (career pages + boards).
2. **Deduplicate** and store in the database: the same “company + role title” doesn’t get multiplied.
3. **Score & filter** using quality rules (work format/location/freshness/relevance).
4. **Generate materials:** for every strong match — outreach texts and notes.
5. **Store results:** everything lives in a database with clear statuses and run history.

## What this project demonstrates about my skills

In short - **I built an end-to-end product-grade service**:

- **Product thinking:** I designed the process as a funnel with quality rules, statuses, and predictable outcomes.
- **Engineering:** web UI + API + job queue + worker + database (not “a script on my laptop”).
- **Data/Automation:** solid data model, deduplication, repeatable runs, scheduling, execution control.
- **Outcome communication:** the output is not raw links — it’s a structured set you can actually act on.

And yes — I **intentionally didn’t automate one-click auto-applying**. If I did, I’d be contributing to the accelerated collapse of the hiring market (and, surprisingly, I still believe in it) 🙂
